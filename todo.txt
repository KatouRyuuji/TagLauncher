  问题 2：能否在应用里做新应用？

  结论：能做一个"寄生"在面板里的功能型小应用，但不能做独立完整的应用。

  可行路径：

  // 理论上 mod 可以这样做：
  api.createPanel("my-app", {
    position: "floating",
    width: 800, height: 600
  }).then(handle => {
    // 向容器写入完整 HTML + 内联事件
    handle.container.innerHTML = `<div id="app-root">...</div>`;

    // 访问应用所有数据作为数据源
    const items = await api.getItems();

    // 自己构建独立的 UI 逻辑
    const state = { ... };
    function render() { ... }
  });

  本质限制：

  ┌─────────────────────────┬───────────────────────────────────┐
  │          需求           │               现状                │
  ├─────────────────────────┼───────────────────────────────────┤
  │ 独立 UI 容器            │ ✅ 面板提供 HTMLElement 容器      │
  ├─────────────────────────┼───────────────────────────────────┤
  │ 数据访问                │ ✅ 完整读写所有数据               │
  ├─────────────────────────┼───────────────────────────────────┤
  │ 新路由/页面             │ ❌ 无路由注入点                   │
  ├─────────────────────────┼───────────────────────────────────┤
  │ 调用系统 API            │ ❌ 无法新增 Tauri 命令            │
  ├─────────────────────────┼───────────────────────────────────┤
  │ 独立生命周期            │ ⚠️ 面板关闭即销毁，无持久化渲染树 │
  ├─────────────────────────┼───────────────────────────────────┤
  │ 加载外部框架(React/Vue) │ ✅ 技术上可行，但体积和冲突问题大 │
  ├─────────────────────────┼───────────────────────────────────┤
  │ Worker/多线程           │ ✅ new Worker() 不受限制          │
  └─────────────────────────┴───────────────────────────────────┘

  ---
  总结

  ┌────────────────────────┬────────┬────────────────────────────────────┐
  │          目标          │ 可行性 │              关键缺口              │
  ├────────────────────────┼────────┼────────────────────────────────────┤
  │ 完全重做外观           │ 60-80% │ 缺组件插槽，DOM 结构无法重组       │
  ├────────────────────────┼────────┼────────────────────────────────────┤
  │ 做新应用（数据工具型） │ 70%    │ 面板容器可用，但无路由、无系统 API │
  ├────────────────────────┼────────┼────────────────────────────────────┤
  │ 做新应用（独立完整型） │ 20%    │ 太多依赖无法满足，需魔改应用本身   │
  └────────────────────────┴────────┴────────────────────────────────────┘

  根本原因：当前架构是"扩展点"模式而非"宿主平台"模式 —— 暴露了数据和通知能力，但没有暴露渲染管道控制权和平台 API
  注册能力。想达到 Obsidian/VSCode 插件那种级别，需要增加组件插槽系统 + Tauri 命令动态注册能力。
